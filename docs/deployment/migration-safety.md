# 운영 DB migration 안전 규칙

Flyway migration은 애플리케이션 이미지와 별도로 데이터베이스에 남는다. 따라서 애플리케이션을 직전 이미지로 롤백해도 이미 적용된 스키마는 자동으로 돌아가지 않는다.

## 필수 규칙

1. 최초 운영 배포 이후 적용된 migration 파일 수정 금지. 변경이 필요하면 더 높은 버전의 새 migration을 추가한다.
2. 파괴적 변경은 한 번에 배포하지 않고 `컬럼 추가 → 코드 전환 → 구 컬럼 제거` 순서의 최소 세 배포로 나눈다.
3. 각 배포의 스키마는 새 코드가 요구하는 구조인 동시에 이전 코드가 동작할 수 있는 구조여야 한다.
4. 컬럼 제거·이름 변경·타입 축소 전에는 직전 이미지로 롤백 가능한지 리뷰에서 확인한다.
5. 배포 워크플로가 성공한 Cloud SQL 백업을 찾지 못하면 migration 실행 전 배포를 중단한다.
