# FE 에이전트와의 계약 동기화

> FE는 별도 에이전트가 **동시에** 개발한다. 두 쪽이 서로를 기다리지 않게 하는 것이 이 문서의 목적이다.

---

## 1. 역할 경계

| 항목 | 백엔드 | 프론트엔드 |
|---|:---:|:---:|
| Kakao Local REST (검색·주소·좌표) | ✅ 서버에서만 | ❌ 브라우저에서 금지 |
| Kakao Maps JavaScript (지도 표시) | ❌ | ✅ 브라우저가 쓰는 유일한 외부 API |
| ODsay / TMAP | ✅ 서버에서만 | ❌ |
| 외부 지도 링크 생성 | ❌ (서버 API 없음) | ✅ 저장된 이름·좌표로 클라이언트가 생성 |
| 사용자 텍스트 렌더링 안전 | 저장·반환만 (일반 텍스트) | ✅ HTML로 실행하지 않을 책임 |

현재 FE는 `frontend/src/api/`의 공용 axios 인스턴스를 쓰고 `baseURL`은 `VITE_API_BASE_URL`에서 읽는다. **백엔드는 이 경로 규약을 깨지 않는다.**

## 2. 백엔드가 FE에 제공해야 하는 것

| 산출물 | 시점 | 방법 |

## 3. 단계별 "FE가 연동 가능해지는 시점"

FE는 아래 시점부터 해당 화면을 실제 서버에 붙일 수 있다. **그 전에는 mock으로 진행**한다.

| 백엔드 단계 완료 | FE가 붙일 수 있는 화면 |
|---|---|
| P1 | 보드 생성, 초대 링크, 참여, 참여자 목록, 내 정보/출발지 |
| P2 | 장소 검색·선택, 지도 마커, 장소 목록/상세/삭제 |
| P3 | 댓글, 투표 생성·참여·결과 |
| P4 | 코스 편집(순서 변경), 코스 확정, 공개 일정 페이지 |
| P5 | 개인 출발 안내 (계산 버튼 + 상태 표시) |
| P6 | 지역 찾기 (진행률 폴링 + 결과 지도) |
| P7 | 주변 장소 검색 (공통 영역 밖 좌표에서 장소 조회), 최종 장소 선택 |

## 4. 오해하기 쉬운 계약 6가지 (FE에 반드시 전달)

1. **코스 초안은 낙관적 잠금이다.**
   `GET`의 `ETag: "draft-{n}"`을 그대로 `If-Match`에 넣어 `PUT`한다. 헤더를 빼면 `400`, 값이 낡으면 `412 VERSION_MISMATCH`(응답에 최신 ETag 포함) → 재조회 후 재시도 UI가 필요하다.

2. **지역 찾기는 즉시 응답이 없다.**
   `202` + `jobId` → **2초 간격 폴링**. SSE·웹소켓·푸시는 MVP에 없다. `progress.phase`로 진행 단계를 표시한다.

3. **출발 안내에는 job ID가 없다.**
   `POST` 후 `GET /departure-guide`를 폴링한다. 상태 6종(`NOT_REQUESTED`/`CALCULATING`/`READY`/`STALE`/`UNAVAILABLE`/`FAILED`)을 모두 화면에 정의해야 한다. `STALE`은 "다시 계산" 유도.
   **"대중교통 경로 없음"도 HTTP 오류가 아니라 `UNAVAILABLE` 상태로 온다.** 계산이 백그라운드에서 돌기 때문이다. `catch`가 아니라 상태 분기로 처리해야 한다.

4. **참여 토큰은 두 번만 내려온다.**
   보드 생성 응답, 참여 응답. 분실하면 복구 불가이며 새 참여자로 다시 들어와야 한다. FE가 저장 책임을 진다.

5. **타인의 출발지는 `registered` 불리언만 온다.**
   좌표·라벨 키 자체가 응답에 없다. 지도에 타인 출발지를 그릴 수 없다.

6. **코스 초안의 스톱에는 `placeDeleted`가 있다.**
   `GET /course-draft`는 확정과 달리 장소 삭제를 막지 않는다. 저장된 스톱이 참조하는 장소가
   나중에 삭제돼도 그 스톱을 숨기지 않고 그대로 반환하되 `placeDeleted: true`로 표시한다.
   FE는 이 값이 `true`인 스톱을 시각적으로 구분(예: 흐리게 표시)하고, 그대로 확정을 시도하면
   `POST /courses`가 `400 INVALID_ARGUMENT`로 거부한다는 점을 안내해야 한다. `legs`는 삭제
   여부와 무관하게 항상 전체 스톱 기준으로 계산되므로 별도 처리가 필요 없다.

## 5. 페이지네이션·목록 규약

- `page`는 **1부터**, `size` 기본 20 / 최대 50
- 응답: `{items, page:{number,size,totalItems,totalPages}}`
- **참여자 목록과 검색 후보는 페이지네이션하지 않는다** (`items`만)

---

## 6. 변경 시 프로토콜

- API 계약을 바꿔야 한다면 **구현 먼저 바꾸지 않는다.** `docs` repo 스펙 갱신 → FE 에이전트에 통보 → 양쪽 반영
- 백엔드가 필드를 **추가**하는 것은 FE를 깨지 않으므로 통보만 한다
- 필드 **삭제·이름 변경·상태값 변경**은 반드시 사전 합의

### 통보 이력

- **2026-07-23**: `GET /boards/{boardId}/course-draft` 응답의 각 stop에 `placeDeleted`(boolean) 필드 추가. 코드 리뷰(PR #11)에서 지적된 "삭제된 장소를 참조한 초안 스톱을 GET에서 그대로 노출하지만 표시 구분이 없다"는 문제의 해결책. 필드 추가이므로 사전 합의 없이 통보만 하며, API명세서_v1.1.md 10.1절과 위 4-6항에 반영했다.
